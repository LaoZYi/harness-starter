# 编写实现计划

根据设计文档（或用户描述），生成一份**零上下文可执行**的实现计划。目标：任何人拿到这份计划，不需要额外了解代码库就能按步骤实现。

当前项目：`Agent Harness Framework`（cli-tool / python）
测试命令：`python -m unittest discover -s tests -v`
检查命令：`python scripts/check_repo.py`

## 输入来源

按优先级查找计划输入：
1. `docs/superpowers/specs/` 下最新的设计文档
2. `.agent-harness/current-task.md` 中的任务描述
3. 用户在对话中直接描述的需求

如果没有找到任何输入，提示用户先运行 `/brainstorm` 或直接描述需求。

## 计划结构

### 计划头部

```markdown
# 实现计划：<标题>

**目标**：一句话说明要达到的效果
**架构**：涉及的模块和它们的关系
**技术栈**：使用的语言、框架、工具
**设计文档**：<链接到 specs 文件>（如有）
```

### 任务列表

每个任务是一个 2-5 分钟可完成的原子步骤。使用 checkbox 语法：

```markdown
## 阶段 1：<阶段描述>

- [ ] 任务 1.1：<动作描述>
  - 文件：`<完整文件路径>`
  - 操作：<具体代码变更，给出完整代码块>
  - 验证：`python -m unittest discover -s tests -v` 或具体检查命令
  - 预期结果：<明确的通过标准>

- [ ] 任务 1.2：...
```

### 任务粒度要求

遵循 TDD 节奏，每个功能点拆分为：

1. **写失败测试** — 完整的测试代码（不是伪代码）
2. **确认测试失败** — 运行 `python -m unittest discover -s tests -v`，确认失败原因正确
3. **实现功能** — 完整的实现代码
4. **确认测试通过** — 运行 `python -m unittest discover -s tests -v`，无回归
5. **提交** — 原子 commit，消息说明做了什么

### 代码块要求

**禁止出现以下内容：**
- `// TODO: ...`
- `// 待实现`
- `/* TBD */`
- `...` 省略号代替真实代码
- "参考现有实现" 而不给出具体代码

每个代码块必须是**可直接复制粘贴到文件中的完整代码**。如果代码太长，拆成更小的任务。

### 每个步骤必须包含

1. **精确的文件路径**（从项目根目录开始的相对路径）
2. **完整的代码变更**（新增 / 修改 / 删除）
3. **验证命令**（运行什么命令确认步骤完成）
4. **预期输出**（命令应该输出什么）

## 自审清单

写完计划后，逐条检查：

- [ ] 每个任务都有文件路径
- [ ] 每个任务都有验证命令
- [ ] 代码块没有占位符
- [ ] 任务之间没有隐含依赖（每个任务显式列出前置条件）
- [ ] 计划与设计文档一致（如有设计文档）
- [ ] 总任务量合理（一个功能通常 10-30 个步骤）

## 输出位置

计划写入以下位置之一：
- `docs/superpowers/specs/YYYY-MM-DD-<topic>-plan.md`
- `.agent-harness/current-task.md`（如果是短期任务）

写入后告知用户计划路径，并提示可以开始按步骤执行。
