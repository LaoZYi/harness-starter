# 任务管理助手

创建、分级和跟踪结构化工作项，集成 `.agent-harness/current-task.md` 进行持久化管理。

项目：`Agent Harness Framework`（cli-tool / python）
测试命令：`make test`

## 适用场景

- 将大任务拆解为可执行的子任务
- 将 code review 反馈转化为可操作项
- 记录开发过程中发现的技术债务
- 跨会话需要持久化的工作追踪

## 第 1 步：创建任务项

向 `.agent-harness/current-task.md` 中写入结构化任务条目。每个条目格式：

```markdown
### #N - [状态] 任务描述

- **优先级**：P1 / P2 / P3
- **验收标准**：完成的具体条件
- **依赖**：blocked-by: #M（如有）
- **工作记录**：（完成时填写）
```

### 优先级定义

| 等级 | 含义 | 处理时机 |
|------|------|----------|
| **P1** | 阻塞性问题，影响核心流程 | 必须立即处理 |
| **P2** | 重要功能或改进 | 当前迭代内完成 |
| **P3** | 锦上添花，非关键项 | 可延后到下一迭代 |

### 状态标记

- `[待办]`：尚未开始
- `[进行中]`：正在处理
- `[已完成]`：已完成并验证
- `[已阻塞]`：被其他任务阻塞

## 第 2 步：任务分级

对 `.agent-harness/current-task.md` 中的待办项进行评估和排序：

1. **阅读所有待办项**，掌握全局
2. **评估优先级**，考虑以下因素：
   - 是否阻塞其他任务？→ 优先级提升
   - 是否面向用户？→ 优先级提升
   - 是否涉及安全风险？→ 强制 P1
3. **重新排序**：按 P1 → P2 → P3 排列
4. **检查 `.agent-harness/lessons.md`**：查看是否有历史教训与当前任务相关，避免重蹈覆辙

## 第 3 步：依赖管理

- 使用 `blocked-by: #N` 标记依赖关系
- 开始任务前，**先验证其所有依赖项已完成**
- 发现循环依赖时立即报告并协助重新拆分
- 被阻塞任务状态标记为 `[已阻塞]`，解除后改回 `[待办]`

## 第 4 步：完成任务

1. **核对验收标准**：逐条确认是否满足
2. **运行测试**：`make test`
3. **更新状态**：改为 `[已完成]`，附简要工作记录
4. **检查连锁影响**：是否有被该任务阻塞的项可以解除？

## 第 5 步：任务归档

当所有任务项均已完成：

1. **最终验证**：运行 `make test` 确认无回归
2. **生成摘要**：任务数量、关键成果、耗时估算
3. **追加日志**：写入 `.agent-harness/task-log.md`
4. **提炼经验**：更新 `.agent-harness/lessons.md`
5. **清理**：将 `current-task.md` 恢复为空模板状态

## 与 agent-harness 的集成

- **任务存储**：所有任务写入 `.agent-harness/current-task.md`，不创建额外的 todo 目录
- **完成归档**：摘要追加到 `.agent-harness/task-log.md`
- **经验参考**：创建任务时检查 `.agent-harness/lessons.md` 中的历史教训
- **生命周期**：遵循 `.claude/rules/task-lifecycle.md` 中定义的任务流转规则

## 一句话总结

**创建 → 分级 → 追踪依赖 → 验证完成 → 归档。每个任务都有明确的验收标准和优先级。**
