# 并行代理调度

针对多个独立问题域，分别派遣专注的代理并行工作，最后统一审查和集成。

## 适用场景

- 存在 3 个以上来自不同根因的失败
- 需要修复多个独立子系统
- 各问题域之间无共享状态或相互依赖
- 项目：Agent Harness Framework（cli-tool，python）

## 不适用场景

- 失败之间存在因果关联（先修根因）
- 需要全局系统理解才能定位问题
- 多个代理可能修改同一文件或同一模块
- 问题数量少于 3 个（直接逐个处理更高效）

## 执行流程

### 第一步：识别独立问题域

1. 分析所有失败或待处理项
2. 按根因和影响范围分组
3. 验证各组之间的独立性：
   - 不修改相同文件
   - 不依赖相同的可变状态
   - 修复一个不会影响另一个的测试结果
4. 如果无法确认独立性，**不要并行**

### 第二步：创建聚焦任务

为每个代理编写自包含的任务描述，包含：

- **范围限定**：明确哪些文件和模块属于本任务，哪些禁止修改
- **完整上下文**：相关的错误信息、日志、堆栈跟踪
- **验证标准**：如何确认修复成功
- **测试命令**：`make test`
- **检查命令**：`make check`
- **输出规格**：修改了哪些文件、修复了哪些问题、是否有残留风险

### 第三步：并行派遣

- 每个问题域派遣一个代理
- 代理之间不直接通信，但共享一个**发现文件**（见下文）
- 设定合理的超时时间

**共享发现文件**：在 `.agent-harness/` 下创建 `dispatch-findings.md`，每个代理完成后将以下信息追加到该文件：
- 修改了哪些文件
- 发现了哪些额外问题（超出本任务范围的）
- 有无残留风险或对其他域的影响

这使得第四步集成审查时有完整的信息，而非仅依赖代理的最终状态。

### 第四步：审查与集成

所有代理完成后，调度者读取 `dispatch-findings.md` 并执行：

1. **冲突检测**：根据发现文件中的修改文件列表，检查是否有重叠
2. **交叉影响分析**：根据发现文件中的"对其他域的影响"，评估是否存在间接冲突
3. **交叉验证**：运行 `make test` 确保所有修复共存时仍然正常
4. **集成测试**：运行 `harness` 确认整体功能
5. **解决冲突**：如有冲突，手动合并并重新验证
6. **清理**：删除 `dispatch-findings.md`

## 有效的任务描述模板

```markdown
## 代理任务：[问题域名称]

### 问题描述
[具体的错误现象和复现步骤]

### 范围
- 可修改：[文件列表]
- 禁止修改：[文件列表]

### 验收标准
- [ ] [具体的通过条件]
- [ ] make test 相关测试通过

### 上下文
[错误日志、堆栈跟踪等]
```

## 核心规则

- **独立性优先**：无法确认独立时，退回串行处理
- **范围隔离**：每个代理的可修改范围不能重叠
- **自包含上下文**：不依赖代理自行探索项目结构
- **集成验证**：并行结果合并后必须重新验证全局一致性
- **失败处理**：单个代理失败不影响其他代理，但集成前必须解决所有失败

## 安全约束

### 工具黑名单

并行代理**禁止**执行以下操作：
- 再次派遣子代理（禁止递归）
- 向用户提问（由调度者统一协调）
- 写入 memory/lessons（由调度者统一沉淀）
- 执行 git push、关闭 Issue 等远端操作
- 修改范围外的文件（严格遵守任务描述中的"可修改"列表）

### 深度限制

最多 1 层并行：调度者 → 并行代理 → **禁止再派遣**。

### 预算共享

所有并行代理共享总预算。每个代理分配独立预算份额（总预算 / 代理数），耗尽时返回当前进度。单个代理超时不阻塞其他代理。

## 状态追踪

```markdown
## 并行调度 [日期]

```

## 子 agent 日志隔离（Issue #14）

并发写 current-task.md 会互相覆盖。每个派出的子 agent 必须使用独立 diary：

```bash
harness agent init agent-1         # 创建 .agent-harness/agents/agent-1/
harness agent diary agent-1 "扫描 src/auth/ 找到 3 处 TODO"
harness agent status agent-1 "正在重写 auth/session.py"
```

主 agent 收集时：

```bash
harness agent list                 # 看哪些子 agent 活跃
harness agent aggregate            # 汇总所有 diary
harness agent aggregate agent-1 agent-3  # 只汇总特定 id
```

id 规范：`^[a-z0-9][a-z0-9-]{0,30}$`。子 agent **禁止**直接写 current-task.md / task-log.md / lessons.md — 主 agent 根据 aggregate 结果决定哪些值得进主档。
