# 并行代理调度

针对多个独立问题域，分别派遣专注的代理并行工作，最后统一审查和集成。

## 适用场景

- 存在 3 个以上来自不同根因的失败
- 需要修复多个独立子系统
- 各问题域之间无共享状态或相互依赖
- 项目：Agent Harness Framework（cli-tool，python）

## 不适用场景

- 失败之间存在因果关联（先修根因）
- 需要全局系统理解才能定位问题
- 多个代理可能修改同一文件或同一模块
- 问题数量少于 3 个（直接逐个处理更高效）

## 执行流程

### 第一步：识别独立问题域

1. 分析所有失败或待处理项
2. 按根因和影响范围分组
3. 验证各组之间的独立性：
   - 不修改相同文件
   - 不依赖相同的可变状态
   - 修复一个不会影响另一个的测试结果
4. 如果无法确认独立性，**不要并行**

### 第二步：创建聚焦任务

为每个代理编写自包含的任务描述，包含：

- **范围限定**：明确哪些文件和模块属于本任务，哪些禁止修改
- **完整上下文**：相关的错误信息、日志、堆栈跟踪
- **验证标准**：如何确认修复成功
- **测试命令**：`python -m unittest discover -s tests -v`
- **检查命令**：`python scripts/check_repo.py`
- **输出规格**：修改了哪些文件、修复了哪些问题、是否有残留风险

### 第三步：并行派遣

- 每个问题域派遣一个代理
- 代理之间不通信，各自独立完成
- 设定合理的超时时间

### 第四步：审查与集成

所有代理完成后：

1. **冲突检测**：检查是否有多个代理修改了同一文件
2. **交叉验证**：运行 `python -m unittest discover -s tests -v` 确保所有修复共存时仍然正常
3. **集成测试**：运行 `PYTHONPATH=src python -m agent_harness.cli` 确认整体功能
4. **解决冲突**：如有冲突，手动合并并重新验证

## 有效的任务描述模板

```markdown
## 代理任务：[问题域名称]

### 问题描述
[具体的错误现象和复现步骤]

### 范围
- 可修改：[文件列表]
- 禁止修改：[文件列表]

### 验收标准
- [ ] [具体的通过条件]
- [ ] python -m unittest discover -s tests -v 相关测试通过

### 上下文
[错误日志、堆栈跟踪等]
```

## 核心规则

- **独立性优先**：无法确认独立时，退回串行处理
- **范围隔离**：每个代理的可修改范围不能重叠
- **自包含上下文**：不依赖代理自行探索项目结构
- **集成验证**：并行结果合并后必须重新验证全局一致性
- **失败处理**：单个代理失败不影响其他代理，但集成前必须解决所有失败
