# 编写新技能文档

你的任务是用 TDD 方法论编写一个高质量的 Claude Code 技能（skill）文档。

项目：`Agent Harness Framework`（cli-tool，python）

## 铁律：没有失败测试就没有技能

**绝不凭空编写技能。** 必须先证明「没有这个技能时，AI 会犯什么错」，然后才动笔。

## RED-GREEN-REFACTOR 流程

### RED 阶段：证明需要这个技能

1. **定义场景**：列出 3-5 个该技能应覆盖的典型任务场景
2. **无技能测试**：在没有该技能的情况下，让 AI 执行这些场景
3. **记录失败**：精确记录每个场景中的具体失败表现：
   - 遗漏了什么步骤？
   - 做了什么危险操作？
   - 输出质量差在哪里？
4. 将失败记录整理成「失败证据表」：

```
| 场景 | 期望行为 | 实际行为 | 问题分类 |
|------|---------|---------|---------|
| ...  | ...     | ...     | 遗漏/错误/低质量 |
```

如果无法证明存在失败，**停止**——不需要这个技能。

### GREEN 阶段：编写最小可行技能

编写 SKILL.md 文件，结构如下：

```markdown
# <技能名称>

## 使用时机

<描述「什么时候」使用，而非「做什么」。用触发条件描述，不用流程摘要。>

## 概述

<核心原则——一句话说明这个技能最重要的一条规则。>

## 快速参考

<精简的操作步骤清单，供熟练者快速查阅。>

## 详细流程

<完整的分步指引，每步包含具体命令或检查项。>

## 示例

<一个完整的、从头到尾的优秀示例。不是片段，是完整流程。>

## 常见错误

<列出 3-5 个常见错误及正确做法。>
```

关键写作原则：
- **「使用时机」必须是触发条件**，不是流程摘要。对比：
  - 错误："本技能帮助你完成分支管理"
  - 正确："当你完成了一个功能分支的开发，需要合并或清理时"
- **示例必须完整**——从最初状态到最终结果，包含所有中间步骤
- 用 `make test` 和 `make check` 作为验证命令

### REFACTOR 阶段：封堵漏洞

1. **构建合理化借口表**：列出 AI 可能用来跳过技能步骤的借口

```
| 借口 | 为什么不成立 | 强制规则 |
|------|------------|---------|
| "这次情况简单，不需要..." | 简单情况更容易出错 | 必须执行完整流程 |
| "用户说了可以跳过" | 用户可能不了解风险 | 警告风险后再确认 |
```

2. **红旗清单**：列出必须立即停止的危险信号
3. **明确禁止的变通方案**：把常见的偷懒路径写成禁止规则
4. **重新验证**：用 RED 阶段的场景重新测试，确认所有失败都已修复

## 技能文件放置

- 命令文件：`.claude/commands/<技能名>.md`
- 如果用了模板变量，扩展名为 `.md.tmpl`
- 文件名用小写短横线格式（kebab-case）

## 质量检查清单

写完后逐项检查：

- [ ] 「使用时机」是触发条件而非流程摘要
- [ ] 有且仅有一个核心原则
- [ ] 示例是完整流程而非片段
- [ ] 常见错误至少 3 条
- [ ] 合理化借口表至少 3 行
- [ ] 红旗清单已列出
- [ ] 用 RED 阶段场景重新验证通过
