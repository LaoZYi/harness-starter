# 错误归因规则

用户指出问题或自检发现不符时，禁止只改代码就完事。必须同时完成以下记录：

## 必做步骤

1. **task-log 返工记录**：在 `.agent-harness/task-log.md` 追加返工条目
   - 用户反馈：原话或概括
   - 根因：为什么出错
   - 修复了什么：具体改动
   - 改了哪些文件
2. **lessons 教训提炼**：在 `.agent-harness/lessons.md` 追加一条
   - 错误：发生了什么
   - 根因：为什么会发生
   - 规则：以后怎么避免
3. **补规则（如需要）**：如果错误是因为缺少约束导致的，在 `AGENTS.md` 硬规则里新增一条

## 判断标准

以下情况必须触发归因：
- 用户说"不对"、"改回去"、"这不是我要的"
- 测试失败后发现是自己引入的 bug
- 发现之前的改动破坏了已有功能
