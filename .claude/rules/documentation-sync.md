# 文档同步规则

改了代码就必须改文档，禁止堆到任务结束批量更新。

## 映射表

| 代码变更类型 | 必须更新的文档 | 更新什么 |
|---|---|---|
| 新增、修改、移除功能 | `docs/product.md` | 功能列表和完成标准 checkbox |
| 改了文件结构或模块边界 | `docs/architecture.md` | 文件结构、模块职责 |
| 改了运行、测试、部署方式 | `docs/runbook.md` | 对应命令和步骤 |
| 改了协作约束或硬规则 | `AGENTS.md` | 硬规则列表 |

## 执行要求

- 每完成 **1 个功能点**就立即更新，不是做完所有功能再更新
- 如果一次改动涉及多个文档，全部更新后再进入下一个功能点
- `docs/product.md` 中的 checkbox：实现完成改为 `[x]`，发现不可行改为 `[-]` 并注明原因
