# 安全规则

- 禁止提交包含密钥、token 或密码的文件（.env, credentials.json 等），因为一旦进入 git 历史就无法彻底清除。
- 禁止执行 `rm -rf`、`git reset --hard`、`git push --force` 等不可逆操作，除非用户在当前对话中明确要求。因为这些操作无法撤销，误操作代价极高。
- 禁止直接修改生产环境的数据库、配置或部署状态。所有生产变更必须通过代码提交和正常部署流程。
- 数据敏感级别：standard。级别为 high 时，禁止在日志、错误信息或注释中输出用户数据。
- 新增依赖前确认其必要性。禁止为单次使用引入重量级依赖，因为每个依赖都是长期维护负担和供应链风险。
