# 贡献指南

这个仓库把协作规则收敛到仓库内文件，而不是分散在聊天记录里。

开始前请按顺序阅读：

1. `AGENTS.md`
2. `docs/workflow.md`
3. 任务相关的 `docs/` 文档

## 标准流程

1. 用一句话描述本次改动目标。
2. 先做最小实现。
3. 为行为变化补测试。
4. 为规则变化补文档。
5. 运行 `make check`。
6. 如果改了行为或发布相关内容，运行 `make ci`。

## 提交内容应该包含什么

- 变更摘要：说明改了什么。
- 验证方式：列出你跑过的命令。
- 文档同步：说明更新了哪份文档。
- 风险说明：还有哪些边界没覆盖。

## 不要这样做

- 不要把长期规则只写进某个工具的 memory。
- 不要发明新的隐藏命令。
- 不要只改代码不补文档或测试。

## 发布前阅读

发布相关操作请参考 `docs/release.md`。
