# 发布清单

这个文件描述的是“发布动作前要核对什么”，不是自动化脚本的替代品。

## 适用场景

- 准备打版本。
- 准备合并一组较大的改动。
- 想把这个 starter 用到新仓库前做最后巡检。

## 发布前检查

1. `README.md` 是否仍然准确描述主命令和目录。
2. `AGENTS.md` 是否仍然是短入口，没有膨胀成百科。
3. `docs/product.md` 是否覆盖最新行为。
4. `docs/architecture.md` 是否覆盖最新结构约束。
5. `docs/workflow.md` 是否仍和实际协作流程一致。
6. GitHub 模板是否仍然指向正确文档。
7. `make ci` 是否通过。

## 发布动作建议

1. 总结本次对外可见变化。
2. 记录验证命令和结果。
3. 明确是否需要迁移说明。
4. 标记后续待办，而不是把半成品塞进当前发布。

## 发布后复盘

1. 本次问题里有哪些已经可以前移成脚本检查。
2. 哪些 review 评论重复出现，应该进入文档或模板。
3. 是否需要新增一个更清晰的入口文档。
