# 发布清单

这个框架发布前，重点不是业务正确性，而是“能否稳定初始化别人项目”。

## 发布前检查

1. `make ci` 是否通过。
2. `README.md` 是否仍准确描述初始化流程。
3. `docs/runbook.md` 是否覆盖最新命令。
4. `templates/common/` 是否仍然通用，没有带入样例业务。
5. `presets/` 是否覆盖当前支持的项目类型。
6. 评估脚本、初始化脚本和探测脚本是否都能在临时目录里跑通。

## 发布动作建议

1. 说明本次影响了哪些生成文件或探测字段。
2. 说明是否会影响已有接入项目的升级策略。
3. 如果修改了模板，附上至少一个生成结果摘要。
